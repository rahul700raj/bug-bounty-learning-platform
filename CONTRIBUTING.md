# Contributing to Bug Bounty Learning Platform

Thank you for your interest in contributing! This platform is designed to help people learn about web security and bug bounty hunting.

## How to Contribute

### 1. Adding New Vulnerabilities

To add a new OWASP vulnerability demonstration:

1. Create both vulnerable and secure endpoints in `backend/routes/owasp.js`
2. Add clear comments explaining the vulnerability
3. Include exploitation examples
4. Document the secure alternative

### 2. Improving Documentation

- Fix typos and grammar
- Add more detailed explanations
- Include real-world examples
- Add references to CVEs or bug bounty reports

### 3. Adding Challenges

Create new practice challenges:
- Define clear objectives
- Provide hints
- Include solution writeups
- Test thoroughly

### 4. Frontend Improvements

- Enhance UI/UX
- Add interactive features
- Improve accessibility
- Mobile responsiveness

## Code Style

- Use clear, descriptive variable names
- Comment complex logic
- Follow existing code structure
- Keep functions focused and small

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/new-vulnerability`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

## Vulnerability Reporting

If you find a security issue in the platform itself (not the intentional vulnerabilities):

1. **DO NOT** open a public issue
2. Email: security@example.com
3. Include detailed reproduction steps
4. Allow time for fix before disclosure

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on education and learning
- No malicious code or exploits

## Questions?

Open an issue or reach out to the maintainers.

Thank you for helping make security education accessible!
